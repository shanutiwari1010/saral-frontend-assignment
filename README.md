## Saral Frontend Assignment — Gamification Rewards

A small React app that showcases a **Gamification** setup screen. Users can open a modal to create a reward system based on triggers like **Sales**, **Posts**, or **Onboarded**, and optionally make it **time-bound** with an end date.

---

## Demo flow (what the app does)

- Open the app → it lands on **`/gamification`** by default.
- Click **Enable Gamification** → opens **Create Reward** modal.
- Choose a **Reward event**:
  - **Sales** → requires a sales amount
  - **Posts** → requires post count + period
  - **Onboarded** → no extra fields
- Choose a **Reward type**:
  - **Bonus** → requires bonus amount
  - **Points (Commission tier upgrade)** → choose a commission tier via sub-dialog
  - **Discount** → no extra fields
- Optional: **Make the reward time bound** → requires selecting an end date.
- The **Create Reward** button stays disabled until required inputs are complete and shows tooltips explaining what’s missing.

## Redux architecture (production-friendly)

Redux Toolkit is used with:

- **Async thunk** for submission
  - `submitReward(formValues)` in `src/store/slices/gamificationSlice.ts`
- **Normalized reward storage**
  - `rewardIds: string[]`
  - `rewardsById: Record<string, Reward>`
- **Status-based async state**
  - `submitStatus: "idle" | "loading" | "succeeded" | "failed"`
- **Selectors**
  - `selectIsSubmitting`, `selectGamificationError`, `selectRewards`, etc.

This keeps UI components simpler and makes it easier to extend (editing/deleting rewards, persistence, API integration, etc.).

---

## Forms + validation

`CreateRewardModal` uses:

- React Hook Form for form state
- Zod schema for validation rules (required fields depend on selected event/type and time-bound toggle)

User feedback:

- Inline validation messages (per field)
- Toasts for success/failure
- Inline error banner using slice `error` when present

---

## Styling conventions

- Prefer **semantic Tailwind tokens** over hardcoded colors:
  - `bg-popover`, `text-foreground`, `border-border`, `text-muted-foreground`, etc.
- Avoid hex colors (`#...`) in components.
- Global tokens live in `src/index.css`.
